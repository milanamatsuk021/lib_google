# Описание файлов в проекте "Моя Книжная Полка"

Этот документ описывает назначение каждого файла, находящегося в корневой директории проекта.

### 1. `index.html`

- **Назначение:** Это главный и единственный HTML-файл приложения, который загружается в браузере пользователя.
- **Роль:** Он служит "каркасом" для всего приложения.
- **Ключевые элементы:**
    - `<meta>` теги: Задают кодировку, заголовок страницы и настройки для мобильных устройств.
    - `<script src="https://cdn.tailwindcss.com"></script>`: Подключает CSS-фреймворк **Tailwind CSS**. Именно благодаря ему приложение имеет стильный внешний вид без единого отдельного CSS-файла.
    - `<script type="importmap">`: Определяет, откуда загружать зависимости (библиотеки), такие как `react` и `@google/genai`.
    - `<div id="root"></div>`: Основной контейнер. Это "точка монтирования", куда React вставляет всё приложение.
    - `<script type="module" src="/index.tsx"></script>`: Загружает и запускает главный скрипт приложения.

### 2. `index.tsx`

- **Назначение:** Это "точка входа" для React-приложения.
- **Роль:** Его задача — найти HTML-элемент с `id="root"` и "отрисовать" (рендерить) в нём главный компонент приложения — `<App />`.
- **Что он делает:** Инициализирует библиотеку React и запускает всю логику, описанную в `App.tsx`.

### 3. `App.tsx`

- **Назначение:** Это сердце и мозг всего приложения.
- **Роль:** В этом файле содержится **100% логики и интерфейса** приложения. Он был намеренно сделан единым файлом, чтобы имитировать стиль написания кода новичком и упростить понимание связей между разными частями.
- **Содержимое:**
    - **Определения типов (TypeScript):** Все структуры данных, такие как `Book`, `BookCategory` и т.д.
    - **Компоненты:** Все визуальные блоки (кнопки, иконки, карточки книг, модальные окна, навигация) определены здесь как React-компоненты.
    - **Состояние (State):** Вся информация, которая может меняться (список книг, текущий экран, результаты поиска), хранится здесь с помощью хуков `useState`.
    - **Логика:** Все функции для добавления, удаления, обновления книг, а также для поиска через **Google Books API** и сохранения данных в `localStorage` находятся здесь.
    - **JSX-разметка:** Вся структура HTML, которая отображается пользователю.

### 4. `README.txt`

- **Назначение:** Руководство для разработчика.
- **Роль:** Этот файл не влияет на работу приложения, но служит документацией. Он подробно описывает **внутреннюю архитектуру** файла `App.tsx` — как организован код, за что отвечает каждая функция и компонент. Это помогает быстрее разобраться в логике проекта.

### 5. `metadata.json`

- **Назначение:** Файл с метаданными для хостинг-платформы.
- **Роль:** Содержит базовую информацию о приложении, такую как его название (`name`) и описание (`description`). Эта информация может использоваться платформой для отображения в каталоге приложений или для других служебных целей.

### 6. Удаленные файлы (components, services, hooks, types.ts)

- **Статус:** Эти файлы и папки **отсутствуют** в текущей версии проекта.
- **Причина:** Они были намеренно удалены, а их содержимое было перенесено и объединено в одном файле (`App.tsx`) для достижения максимальной простоты структуры, как если бы проект писал начинающий разработчик.
