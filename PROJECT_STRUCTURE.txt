# Описание файлов в проекте "Моя Книжная Полка"

Этот документ описывает назначение каждого файла, составляющего рабочее приложение.

### 1. `index.html`
- **Назначение:** Главный HTML-файл, служащий каркасом для приложения.
- **Ключевые элементы:** Подключает Tailwind CSS, определяет importmap для библиотек (`react`, `@google/genai`) и содержит контейнер `<div id="root">` для монтирования React-приложения.

### 2. `index.tsx`
- **Назначение:** Точка входа для React-приложения.
- **Роль:** Инициализирует React и рендерит главный компонент `<App />` в `<div id="root">`.

### 3. `App.tsx`
- **Назначение:** Сердце и мозг приложения.
- **Роль:** Управляет состоянием UI, навигацией между экранами и взаимодействием с сервисами. Не содержит прямой логики работы с базой данных или API, а делегирует эти задачи соответствующим сервисам.

### 4. `types.ts`
- **Назначение:** Единый источник истины для всех структур данных.
- **Роль:** Содержит все определения типов TypeScript (`interface`, `enum`, `type`), используемые в приложении, обеспечивая строгую типизацию и предсказуемость кода.

### 5. `services/` (Папка)
- **Назначение:** Содержит модули для взаимодействия с внешними системами и управления данными.

#### 5.1. `dbService.ts`
- **Назначение:** Сервис для работы с браузерной базой данных **IndexedDB**.
- **Роль:** Инкапсулирует всю логику по созданию, чтению, обновлению и удалению (CRUD) записей о книгах. Является единственной точкой взаимодействия с хранилищем данных.

#### 5.2. `googleBooksService.ts`
- **Назначение:** Сервис для выполнения запросов к внешнему API.
- **Роль:** Содержит функцию `searchBooks` для поиска книг через Google Books API.

#### 5.3. `geminiService.ts`
- **Назначение:** Сервис для взаимодействия с генеративным ИИ.
- **Роль:** Содержит функцию `getRecommendationsFromGemini` для получения книжных рекомендаций на основе библиотеки пользователя.

### 6. `components/` (Папка)
- **Назначение:** Содержит переиспользуемые компоненты пользовательского интерфейса.

#### 6.1. `ui/Select.tsx`
- **Назначение:** Универсальный компонент выпадающего списка.
- **Роль:** Создан для устранения дублирования кода и обеспечения единого стиля для всех `<select>` элементов в приложении.

#### 6.2. Остальные файлы в `components/`
- `BookCard.tsx`, `BottomNav.tsx`, `Header.tsx`, модальные окна и т.д. — каждый файл представляет собой отдельный, логически изолированный компонент UI.

### 7. `initial-books.json`
- **Назначение:** Статический файл с данными.
- **Роль:** Содержит массив книг в формате JSON, который используется `dbService` для первоначального заполнения базы данных при первом запуске приложения.

### 8. `README.txt` & `PROJECT_STRUCTURE.txt`
- **Назначение:** Документация проекта.
- **Роль:** Описывают архитектуру, назначение файлов и общую логику работы приложения.

### 9. `metadata.json`
- **Назначение:** Файл с метаданными для хостинг-платформы.
- **Роль:** Содержит название и описание приложения.
